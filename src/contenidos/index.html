Por qué se diferencia Backend y Frontend?

Se separa el Frontend del Backend por un aspecto tecnológico. Lo que sucede es que parte del código lo baja el cliente y otra parte no.

Esto es como cuando bajás un juego online. Tenés que descargarlo e instalarlo, pero necesitás un servidor para poder conectarte.

El hecho de que el cliente baje (y pueda inspeccionar y modificar el código) hace que el código sea más inseguro, por lo que el frontend y el backend se manejan de formas diferentes. Si tenés en tu código contraseñas o claves nunca las deberías poner en el frontend.

El Frontend siempre fue HTML, CSS y Javascript. Para lo que es Backend hay mucha más variedad: PHP, C#, Java y NodeJs entre otros.

Uno entonces pensaría "es más fácil conseguir trabajo si aprendo Frontend". La respuesta es Ni.

No es más fácil porque: Para lo que es Frontend no se buscan empleados "Que sepan Frontend" sino que se buscan por Frameworks o tecnologías. Leasé: "Que tengan experiencia trabajando con: Angular, React, Vue.js, EmberJs, etc".

Pero sí es más fácil porque: Aún si los frameworks son muy diferentes, el lenguage es siempre Javascript. Lo que hace que si ya aprendiste 1 o 2, los demás te deberían parecer muy fáciles, intuitivos y hasta predecibles.

Qué es Frontend?

Una forma muy resumida de responder "Qué es Frontend?" sería "La parte de la programación de websites que abarca HTML, CSS y Javascript."

HTML es el lenguaje que se usa para definir layouts y CSS es lo que se usa para darle estilos y forma a esos layouts.

Para entender un poco mejor esto quiero que piensen en las páginas web como cajas con texto adentro, ya que HTML es básicamente eso. Y CSS toma esos Contenedores y esos Textos y los "embellece" definiendo colores, tamaños, y los posiciona. Por ejemplo: a la izquierda ponemos el Contenedor que será la barra lateral, arriba del todo el nombre de la página y al medio ocupando todo el espacio posible es el contenido principal que quieren que el usuario lea.

Pondré ejemplos de código HTML y CSS más adelante.

"Esto significa que tengo que saber diseñar?"

No es necesario. Si vas a trabajar para una empresa, éstas por lo general tienen diseñadores. Si vas a trabajar por tu cuenta podés contratar diseñadores aparte.

Ahora HTML y CSS son contenedores con texto que están embellecidos. Y qué pasa con Javascript?

Javascript es más complicado. Es la parte lógica de los websites. Un ejemplo sencillo de mostrar esto es cuando te registrás en una nueva página y estás llenando tus datos y en un momento uno de estos campos se pone rojo y dice "Ese nombre de usuario ya está en uso".

Obviando los colores y animaciones lindas, algo disparó ese mensaje. Lo que sucedió fue que al haber escrito el nombre de usuario, una lógica valido si ese nombre de usuario ya existía, descubrió que efectivamente ya estaba en uso y decidió mostrar ese mensajae de error.

Hay muchos más casos, pero la idea es dejar claro que toda "lógico" que tiene un website es generalmente generado por Javascript.
